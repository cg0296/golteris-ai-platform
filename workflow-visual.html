Inside the Black Box

The broker's inbox sits in the middle. Customer conversations flow on the left, carrier conversations on the right. All data passes through email.

Customer Side

1. Read Customer RFQ

Broker opens customer email and mentally extracts origin, destination, truck type, insurance, lane specs, truck count, pricing expectations.

Tools: Outlook / Gmail

2. Check Completeness

Broker checks if all required fields are present based on experience and memory. No checklist, no validation rules.

Tools: Broker's memory

3. Follow Up (if needed)

If info is missing, broker manually writes a follow-up email. Then waits for a reply — could be hours or days.

Tools: Outlook / Gmail

8. Send Final Quote

Tools: Outlook / Gmail

Carrier Side

5. Distribute to Carriers

Broker attaches spreadsheet and composes individual emails to each carrier. May forget one or send wrong version.

Tools: Outlook / Gmail + Excel

6a. Collect & Compare Bids

As bid emails arrive, broker reads each one, extracts pricing, and manually enters into a comparison spreadsheet.

Tools: Outlook / Gmail + Excel

6b. Negotiate & Select

Broker evaluates bids on cost, service fit, and reliability. May go back and forth with carriers via email.

Tools: Outlook / Gmail + judgment
Inside the AI Agent

The email box stays as the interface. The Orchestration Agent reads/writes email and delegates to specialized sub-agents.

Intake Agent

Parses inbound customer RFQ emails. Extracts structured fields via LLM. Validates completeness. Drafts follow-ups if info is missing.

Structuring Agent

Converts validated RFQ data into a standardized carrier-ready spreadsheet template. Zero manual data entry.

Carrier Outreach Agent

Selects carriers from DB by lane, equipment, and history. Distributes the structured RFQ. Tracks response status.

Bid Analysis Agent

Parses carrier bid emails (multi-format). Normalizes pricing into a comparison matrix. Ranks by cost, history, and fit.

Pricing Agent

Applies configurable markup rules to the selected carrier bid. Calculates final sell price.

Quote Agent

Generates the customer-ready quote document. Formats as per-lane, per-truck, or average. Sends via email after approval.

SINGLE POINT OF FAILURE

One person, one inbox. Broker unavailable = everything stops.

UNSTRUCTURED DATA

All information lives in email threads and manually-built spreadsheets. No system of record.

NO AUDIT TRAIL

Can't trace how a quote was built or why a carrier was selected. It's all in the broker's head.

SERIAL THROUGHPUT

One quote at a time. Scaling means hiring more brokers and duplicating the same manual process.

ERROR-PRONE

Every data transfer is manual copy/paste between emails and spreadsheets. Typos, missed fields, wrong versions.

Inside the AI Agent — Multi-Agent Hierarchy

The email box stays as the interface to the outside world. Behind it, an Orchestration Agent delegates work to specialized sub-agents.

Orchestrator Agent

The central coordinator. Receives triggers (new email, bid received, timer), decides which sub-agent to invoke, manages state transitions, and enforces human-in-the-loop checkpoints. Maintains the full quote lifecycle in a structured data store.

ROLE: Routing • State Management • HITL Gates • Audit Log
CUSTOMER-FACING

Intake Agent

Monitors inbound email. Parses natural-language RFQs using LLM. Extracts structured fields (origin, destination, truck type, insurance, lanes, count, pricing format). Validates completeness against rules. If missing info, drafts follow-up email for approval.

Tech: LLM • Email API • Validation Engine
DATA

Structuring Agent

Takes validated RFQ data and generates a standardized carrier-ready spreadsheet. Normalizes lane details, equipment specs, insurance requirements into a consistent template. Zero manual data entry.

Tech: Template Engine • Data Normalization
CARRIER-FACING

Carrier Outreach Agent

Selects carriers from database based on lane, equipment type, and historical performance. Distributes the structured RFQ to matched carriers via email or API. Tracks which carriers were contacted and response status.

Tech: Carrier DB • Matching Algorithm • Email API
CARRIER-FACING

Bid Analysis Agent

Monitors inbox for carrier bid responses. Parses pricing from emails and spreadsheets (multiple formats). Normalizes into a comparison matrix. Ranks carriers by cost, service history, and operational fit. Recommends best bid.

Tech: LLM • Email Parsing • Scoring Model
CUSTOMER-FACING

Pricing & Quote Agent

Applies configurable markup rules to selected carrier bid. Generates the final customer-ready quote in the requested format (per lane, per truck, or average). Composes and sends the quote email after broker approval.

Tech: Pricing Engine • Doc Generation • Email API
HUMAN-IN-THE-LOOP

Broker Dashboard

Not an agent — the human oversight layer. Broker sees real-time quote status, AI recommendations, and approval gates. Can approve, edit, or reject at any checkpoint. Full audit trail of every decision.

Interface: Web Dashboard • Notifications
