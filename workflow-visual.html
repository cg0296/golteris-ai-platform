Golteris AI Platform

Operational AI for freight brokerage. Replacing manual inbox-driven processes with an intelligent, auditable pipeline.

How the Agent System Runs

Agents never run on their own. A human starts them, a human stops them, and a human can see exactly what is being worked on at all times. No rogue agents, no silent autonomy, no surprises.

Human-Controlled by Default

Every workflow has an explicit on/off switch. Users enable the agents they want running, pause them anytime, and see a live view of what's in progress. The platform is designed for operators who need to trust it, not replace them.

Explicit Start

A user turns each workflow on. Nothing runs until a human enables it. No hidden background jobs.

Explicit Stop

A user can pause or kill any workflow instantly. In-flight runs finish cleanly and the pipeline stops.

Live Visibility

A dashboard shows every active workflow, every in-progress run, every step the system is taking right now.

What's Happening Under the Hood

When a user enables a workflow, the system spins up the necessary triggers and stands up short-lived agents that wake up, do one job, write their result to the database, and exit. The database is the brain — everything else is ephemeral. That's how reliability, auditability, and cost control come for free.

The Three Layers

Triggers

Things that wake work up: cron timers, webhooks, queue messages, DB changes, human clicks, scheduled delays.

Orchestrator

Reads current workflow state, decides what runs next, enqueues the appropriate agent. Stateless itself.

Agents

Focused, single-purpose functions. Read input, do one job, write output to the database, exit.

↓   all three write to   ↓

Database

The single source of truth for every workflow run, every step, every artifact, every decision. Nothing "remembers" anything — the database does.

What Wakes an Agent Up

Once a user enables a workflow, these are the only things that can cause an agent to run. Every trigger is bounded by the workflow's on/off state — if it's off, nothing fires.

Polling on a Schedule

A cron job fires every N seconds while the workflow is enabled. Example: check the mailbox every 60 seconds for new emails.

Webhook / Event Push

An external service calls in when something happens. Example: Gmail push notification on new mail.

Queue Message

A previous step enqueues the next. Example: extraction finishes → queues a "generate quote sheet" job.

Database Change

A new row or status change triggers the next step. Example: status flips to ready_to_quote.

Human Action

A broker clicks approve or replies in a UI. Example: HITL gate approving a draft response.

Timer / Delayed Job

Scheduled for later. Example: "If carrier hasn't replied in 4 hours, send a nudge."

Why It's Built This Way

Reliability

A crashed worker doesn't lose state. The database remembers where the workflow was — a new worker picks it up.

Cost Control

Agents only consume LLM tokens when they run. Idle workflows cost nothing.

Auditability

Every state transition is a row. Every decision is traceable from intake to completion.

Horizontal Scale

Add more workers. No coordination needed — the database is the coordination layer.

Testability

Each agent is a function with clear inputs and outputs. Call it with fake data, assert the result.

Simplicity

Boring orchestration with AI called as a subroutine. No always-on agent fantasy, no hidden state.

An AI-powered operations platform that converts natural-language business communication into structured work, tracked workflows, and downstream actions.

Most businesses still translate unstructured requests into spreadsheets, forms, and workflow systems by hand. This is slow, inconsistent, and impossible to scale. Golteris standardizes that translation layer.

What the Platform Does

Golteris monitors inbound business communication — email, messages, documents — and converts it into structured, actionable data. It then orchestrates the downstream workflow automatically.

  • Accepts natural-language input from any channel
  • Extracts key business information using AI
  • Converts it into a defined data model
  • Generates spreadsheets, records, or outbound content
  • Starts and tracks the related workflow through completion
  • Escalates to humans when confidence is low or risk is high

The Problem It Solves

Across industries, operational teams are buried in email-driven processes. Someone manually reads a request, interprets it, asks follow-up questions, builds a sheet, coordinates with external parties, compares responses, and sends back a result.

  • Slow response times on repeatable work
  • Inbox chaos — requests get buried or lost
  • Inconsistent handling across team members
  • Missed details leading to lost revenue
  • Total dependence on one person remembering the process
  • No audit trail for how decisions were made

Where It Applies

Any workflow where unstructured communication needs to become structured work. The platform is not limited to one industry — it's built for the pattern, not the vertical.

  • Freight brokerage and logistics quoting
  • RFP and procurement response
  • Sales handoffs and deal intake
  • Customer onboarding and intake forms
  • Support requests and case processing
  • Claims processing and compliance workflows
  • Any email-heavy, spreadsheet-driven operation

Why Now

Embedding AI into an organization is quickly shifting from something that creates competitive advantage to something required to stay competitive at all.

  • Businesses that use AI well will reduce manual work, improve efficiency, respond faster, and deliver a better customer experience
  • Businesses that do not will increasingly struggle to keep pace
  • The need is not just to experiment with AI, but to identify where it can be applied in practical ways that improve operations
  • LLM capabilities have reached the point where parsing unstructured communication into structured data is reliable enough for production use

How It Works

The platform uses a multi-agent architecture. Low-level agents handle focused, repeatable tasks. A central orchestrator manages flow, state, and sequencing. AI handles the messy parts — interpreting natural language, fuzzy matching, and exception handling. Deterministic automation handles everything else.

Monitor

Watch inbound channels (email, messages, uploads) for new requests or replies.

Extract

AI parses natural language into structured fields. Flags what's missing.

Validate

Check completeness against business rules. Draft follow-ups automatically if needed.

Generate

Create spreadsheets, records, outbound content, or structured artifacts from extracted data.

Coordinate

Distribute work to external parties, collect responses, and track status across the full lifecycle.

Decide

Compare options, apply business rules, and surface recommendations — with human approval at key gates.

How This Differs From ChatGPT

ChatGPT is a prompt-response assistant — it helps with a single task or conversation. Golteris is an operations system that manages entire business processes from intake to completion. It monitors communication, maintains state across multiple steps and messages, triggers actions automatically, coordinates tools like email and spreadsheets, and uses AI only where ambiguity requires interpretation. The core difference is conversation versus operational execution.

Product Principles

Smallest Repeatable Tasks

Break work into the smallest repeatable units. Give each task a clear input and output.

Explicit State

Keep workflow state explicit. Use orchestration to control the process, not implicit assumptions.

AI Where It Matters

Use AI when reality is messy, not where simple rules are enough. Don't over-automate deterministic steps.

Human Escalation

Escalate to humans when confidence is low or business risk is high. Never let AI silently make high-stakes decisions.

Orchestration First

Low-level agents do focused tasks. A central orchestrator manages flow, state, and which agent runs next.

Auditable by Default

Every decision, every data transformation, every approval is logged. Full traceability from intake to completion.

Inside the AI Agent

The email box stays as the interface. The Orchestration Agent reads/writes email and delegates to specialized sub-agents.

Intake Agent

Parses inbound customer RFQ emails. Extracts structured fields via LLM. Validates completeness. Drafts follow-ups if info is missing.

Structuring Agent

Converts validated RFQ data into a standardized carrier-ready spreadsheet template. Zero manual data entry.

Carrier Outreach Agent

Selects carriers from DB by lane, equipment, and history. Distributes the structured RFQ. Tracks response status.

Bid Analysis Agent

Parses carrier bid emails (multi-format). Normalizes pricing into a comparison matrix. Ranks by cost, history, and fit.

Pricing Agent

Applies configurable markup rules to the selected carrier bid. Calculates final sell price.

Quote Agent

Generates the customer-ready quote document. Formats as per-lane, per-truck, or average. Sends via email after approval.

Case Study

Beltmann Logistics

A national logistics and relocation company drowning in email-driven freight quoting. Their transportation team manually processes every RFQ, carrier bid, and customer quote through a single inbox.

The Pain

Jillian, the transportation lead, described a daily reality of inbox chaos. Her team handles complex project requests with multiple truck types, special equipment, driver unload requirements, and storage charges — all arriving as unstructured emails.

  • High email volume with repetitive back-and-forth
  • Complex RFQs: 6 lift-gate vans + 26 straight trucks + storage in one email
  • Manually retyping email content into spreadsheets
  • Individually emailing carriers for each lane
  • Carrier bids come back as emails, not spreadsheets — broker re-enters everything
  • Client-specific workflows with no standardization
  • Missing details cause delays and lost opportunities

Current Process

Every quote request flows through the team's inbox. There is no system — just email, spreadsheets, and the broker's memory.

  • Customer sends project request via email (natural language)
  • Broker reads email, mentally parses truck types, lanes, requirements
  • Broker follows up for missing info (dates, equipment, unload specs)
  • Broker manually builds carrier pricing spreadsheet
  • Broker emails carriers individually
  • Carriers reply by email — broker extracts pricing manually
  • Broker compares bids, applies markup, sends quote to customer
  • Entire process takes hours to days per project

Proposed Solution

Deploy the Golteris AI platform to monitor Beltmann's mailbox and automate the quoting pipeline. The broker shifts from doing the work to overseeing it.

  • AI agent monitors the inbox and parses inbound RFQs
  • Extracts truck types, lanes, equipment, storage, driver requirements
  • Flags missing fields and drafts follow-up questions automatically
  • Generates carrier-ready spreadsheets from email content
  • Distributes to matched carriers automatically
  • Parses carrier bid responses (email or spreadsheet)
  • Compares bids, applies markup, generates customer quote
  • Supports client-specific workflows — not one rigid process
